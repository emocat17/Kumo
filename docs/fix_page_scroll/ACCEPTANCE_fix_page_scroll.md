# 任务验收报告：优化页面滚动布局

## 1. 问题修复
- **目标**：消除"内外两个滚动条"的现象，实现统一的页面滚动体验。
- **修改内容** (`front/src/layout/MainLayout.vue`)：
    - **容器高度**：将 `.layout-container` 的 `height: 100vh` 改为 `min-height: 100vh`。这允许页面内容自然撑开高度，而不是被限制在视口高度内。
    - **移除内部滚动**：移除了 `.main-wrapper` 的 `overflow: hidden` 和 `.content-area` 的 `overflow-y: auto`。现在主内容区域不再生成独立的滚动条，而是依赖浏览器窗口的滚动条。
    - **侧边栏优化**：为 `.sidebar` 添加了 `position: sticky; top: 0; height: 100vh;`。
        - 效果：当页面向下滚动时，侧边栏会固定在左侧不动，方便导航，同时如果侧边栏内容过多，侧边栏内部仍可独立滚动。
    - **顶部导航优化**：为 `.top-header` 添加了 `position: sticky; top: 0; z-index: 10;`。
        - 效果：页面滚动时，顶部栏也会吸附在顶部，保持随时可见。

## 2. 验证建议
- **滚动测试**：
    - 刷新页面，缩小浏览器窗口高度，或者访问内容较多的页面（如“系统概览”）。
    - 尝试滚动鼠标滚轮。
    - **预期结果**：应该只有一个浏览器右侧的主滚动条，页面整体平滑滚动。侧边栏和顶部栏应该保持固定（吸附）。不再出现内容区域内部的小滚动条。
