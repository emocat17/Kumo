# 任务验收报告：修复底部遮挡与侧边栏折叠按钮问题

## 1. 问题修复
### 1.1 底部遮挡修复
- **原因**：`App.vue` 中设置了 `zoom: 1.1`，导致页面元素放大。此时如果容器高度仍设置为 `100vh`，其实际像素高度会变成视口的 110%，导致底部约 10% 的内容被视口截断（因为 `body` 设置了 `overflow: hidden`）。
- **修复方案**：在 `MainLayout.vue` 中，将 `.layout-container` 的高度调整为 `90.9vh`（即 `100 / 1.1`）。
    - 这样在 `zoom: 1.1` 放大后，容器高度正好等于视口的 100%，从而完美适配屏幕，不再遮挡底部。

### 1.2 侧边栏折叠按钮修复
- **原因**：侧边栏折叠后宽度变为 64px，而顶部的 Logo、文字和折叠按钮挤在一起，宽度超出了 64px，导致布局错乱或按钮被挤出。
- **修复方案**：
    - 修改 HTML 结构：在折叠状态下（`isCollapsed` 为真），隐藏 Logo 和品牌文字。
    - 修改 CSS 样式：添加 `.sidebar.collapsed .sidebar-header { justify-content: center; }`，确保折叠后，折叠按钮居中显示。

## 2. 验证建议
- **底部验证**：
    - 刷新页面，滚动内容区域到底部。
    - 确认现在可以完整看到底部的版权信息 "Kumo . 2025. All rights reserved."，不再被遮挡。
- **侧边栏验证**：
    - 点击侧边栏顶部的收起按钮（`«`）。
    - 确认侧边栏收起后，顶部不再显示 Logo，而是清晰地显示展开按钮（`»`），且按钮居中。
    - 点击展开按钮，侧边栏应正常展开并恢复 Logo 显示。
